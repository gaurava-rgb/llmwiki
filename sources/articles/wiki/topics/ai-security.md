# AI Security

## Summary [coverage: medium -- 2 sources]
Two simultaneous events in late March–early April 2026 crystallized the AI security moment: the Axios npm supply chain attack (March 30, 2026) and Anthropic's accidental leak of Claude Code's source map file. The Axios attack—the third major npm supply chain breach in a year—compromised the package's lead maintainer account, injected a cross-platform remote access trojan (RAT) via a phantom dependency (`plain-crypto-js@4.2.1`), and affected over 100 million weekly npm downloads within 39 minutes across two release branches. The Claude Code leak exposed a 59.8 MB JavaScript source map file from package version 2.1.88 on npm, revealing the internal architecture of a product generating $2.5 billion ARR. These incidents illustrate both faces of AI's security role: AI is a near-term amplifier of exploits, and a long-term solution to the systemic brittleness of software. Anthropic's subsequent announcement of Mythos and Project Glasswing—granting ~50 critical infrastructure providers preview access to a model capable of finding high-severity vulnerabilities in every major OS and browser—is the clearest industry signal yet that AI-powered security is leaving the research phase.

## Key Thesis [coverage: medium -- 2 sources]
The central argument across both sources is that AI will be net negative for security in the short term and net positive in the long term—and that the transition between those two states will be messy, with malicious actors gaining access to exploit-finding capabilities before defensive infrastructure catches up. The Axios attack demonstrates that even hardened ecosystems (post-Shai-Hulud npm reforms, FIDO 2FA, OIDC publishing, granular token caps) fail when a human is the single point of failure—a social engineering attack on maintainer @jasonsaayman bypassed every technical safeguard. The Claude Code leak demonstrates that human error inside AI-native organizations is itself a security surface. Thompson's core thesis: AI is uniquely suited to examine entire codebases and dependency trees at a scale and consistency no human team can match, and will ultimately find and patch vulnerabilities faster than any adversarial actor can exploit them—but not before things get worse.

## Key Insights [coverage: medium -- 2 sources]
- **Supply chain attacks exploit dependency graphs, not source code**: The Axios attacker never modified axios source code. By adding one dependency (`plain-crypto-js@4.2.1`) 18 hours after publishing a clean version to build history, they deployed malware to every downstream consumer. This is the third major npm attack in a year (after Shai-Hulud in September 2025 and PackageGate in January 2026), and each one found vectors that the prior round of hardening failed to close.
- **Human accounts remain the single point of failure**: Post-Shai-Hulud reforms—FIDO 2FA, OIDC trusted publishing, 7-day token caps for new tokens—hardened everything downstream of the maintainer account. They did not harden the account itself. A long-lived classic token (pre-deadline revocation) obtained via social engineering bypassed the entire stack.
- **The Claude Code source map leak was IP exposure, not a data breach**: The 59.8 MB .map file revealed harness architecture, not model weights or customer data. Anthropic called it "human error, not a security breach." The strategic damage is to competitors like Cursor and Microsoft who can now study Anthropic's agentic scaffolding—though legal copying remains prohibited.
- **Mythos has already found thousands of high-severity vulnerabilities**: Anthropic's Project Glasswing blog states Mythos Preview found vulnerabilities across every major OS and browser, including bugs that survived decades of human review and millions of automated security tests. This is the first credible public claim that a frontier AI model has crossed the threshold of outperforming skilled human security researchers on real-world targets.
- **Vibe-coding is a near-term security liability**: Thompson flags that AI-assisted coding will introduce bugs at scale before AI-assisted security review catches up with them. The Anthropic statement's careful framing ("human error, not vibe-coding") reflects awareness that Claude Code itself is a vector for exactly the kind of security debt it is meant to solve.

## Industry Context [coverage: medium -- 2 sources]
The npm ecosystem's vulnerability is structural: the registry hosts millions of packages with deep, often-unreviewed dependency chains, and maintainer accounts are a perennial weak link. The Axios package alone has 100 million+ weekly downloads; compromising it is analogous to a supply chain attack on a major logistics hub. GitHub's post-Shai-Hulud reforms were substantive—npm closed classic token creation, mandated FIDO 2FA, and introduced OIDC trusted publishing—but the Axios attack confirms these measures are necessary but not sufficient while long-lived tokens and social engineering remain viable. Meanwhile, the DARPA Cyber Grand Challenge (2016) established the conceptual benchmark for automated vulnerability discovery; Anthropic's Glasswing post explicitly cites it, claiming Mythos is now "competitive with the best humans" on that class of task—a decade later. OpenAI has faced parallel accusations: a February 2026 House memo cited DeepSeek for using distillation against OpenAI models, mirroring the Anthropic claims against DeepSeek, Moonshot AI, and MiniMax. AI security capability is therefore both a product category and a geopolitical flashpoint.

## Implications [coverage: medium -- 2 sources]
- **All npm-dependent projects should audit long-lived tokens immediately**: The Axios attack vector—a pre-revocation classic token obtained via social engineering—is replicable against any maintainer. Token hygiene (expiration enforcement, eliminating long-lived credentials) is the highest-leverage immediate action.
- **Dependency trees need AI-assisted auditing**: Human review of transitive dependencies is impractical at scale. The attack injected a payload two dependency hops from the target application. Tooling that continuously monitors and validates full dependency graphs (not just direct dependencies) is table-stakes security infrastructure.
- **Organizations should engage Project Glasswing if eligible**: Mythos Preview is being made available to ~50 critical infrastructure providers including Amazon, Microsoft, Apple, Google, and the Linux Foundation. For security teams at large enterprises managing software at scale, early access to AI-powered vulnerability discovery is a genuine defensive advantage before adversaries develop equivalent offensive capability.
- **AI-native companies face a new IP attack surface via build artifacts**: The Claude Code .map file leak came through a packaging error in an npm publish workflow. Any company shipping compiled or bundled code should audit build pipelines for inadvertent inclusion of debug artifacts, source maps, or internal tooling. The fact that Claude Code—an Anthropic-internal security-aware product—shipped this error underscores how universal the risk is.
- **The short-term/long-term security split should inform investment timing**: Thompson's framework argues that AI will make security worse before making it better. Security teams should not defer investment in AI-powered defensive tooling waiting for the "long-term good" phase—adversaries are not waiting. The window between AI becoming capable of finding vulnerabilities and AI being deployed defensively at scale is the highest-risk period, and that window is open now.
