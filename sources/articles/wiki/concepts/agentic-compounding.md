# Agentic Compounding

## Pattern
Agents get better when context, memory, tools, and lessons persist between runs. The corpus treats this as compounding infrastructure: every skill, memory layer, or reusable harness increases the return on the next task.

## Instances
- **2026-04** in [[../topics/claude-code-tools]]: Skills, memory stacks, MCP, and Cowork all point to reusable operating context.
- **2026-04** in [[../topics/ai-agent-workflows]]: Orchestration discussions keep returning to memory and structured state.
- **2026-04** in [[../topics/knowledge-systems]]: Readwise, Obsidian, and knowledge bases externalize memory for later reuse.
- **2026-04** in [[../topics/open-source-ai-builder-stack]]: Builder tools increasingly package reusable capabilities as modular components.
- **2026-04** in [[../topics/builder-feed-notes]]: The short-form feed acts as a discovery loop for the next layer of reusable tools.

## What This Means
The biggest productivity jumps in the corpus come from systems that remember. The lesson is to invest in context persistence and reusable components before chasing isolated prompt wins.
