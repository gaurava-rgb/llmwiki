**the linkedin engagement scraping strategy:**

identify posts from your competitors or industry leaders that are getting significant engagement, use apify linkedin post scraper actor to extract everyone who liked or commented on that post, run company enrichment to get full company details for each person, identify the right decision maker at each company if the engager isn't the right person, reach out referencing their specific engagement on the post

**why this works dramatically better than cold scraping:**

these prospects demonstrated active interest in your solution category within the last 7-30 days, they're actively consuming content about the problem you solve, they're warm to the topic already, and you have perfect context for personalization

**the specific apify actors to use:**

**linkedin profile scraper:** extracts complete profile data including experience, skills, and connections

**linkedin company scraper:** pulls all employee data from company pages

**linkedin post scraper:** extracts all engagers (likes and comments) from specific posts

**linkedin search scraper:** runs sales navigator style searches at scale

## **PART 7: AVOIDING SCRAPING BANS AND DETECTION**

**why most people get blocked:**

scraping too aggressively without rate limiting, using datacenter proxies that platforms easily detect, not rotating user agents and browser fingerprints, accessing platforms from suspicious ip patterns, triggering anti-bot challenges through unnatural behavior

**the proper scraping approach:**

use residential or mobile proxies that look like real user traffic, implement intelligent rate limiting (scrape 100 profiles then pause 30 minutes), rotate user agents randomly between requests, add random delays between actions (2-7 seconds per request), warm up new proxy ips before heavy scraping

**platform-specific considerations:**

**linkedin:** most aggressive anti-scraping, limit to 500-1000 profiles daily per account, use sales navigator legitimate access combined with scraping, never scrape from same ip as your personal linkedin account

**twitter:** easier to scrape than linkedin but still has limits, stay under 5,000 requests daily per ip, use api access when possible combined with web scraping, expect some temporary rate limits, build in retry logic

**instagram:** moderate scraping difficulty, limit to 200-500 profiles daily per account, avoid following/unfollowing which triggers blocks, focus on data extraction not interaction

**youtube:** easiest major platform to scrape, minimal rate limiting on public data, can extract thousands of comments daily, rarely blocks scraping activity

**recovery if you get blocked:**

## switch to fresh proxy immediately, pause scraping from that ip for 24-48 hours, reduce rate limits when resuming, some platforms require captcha solving services, maintain backup proxies ready to rotate
